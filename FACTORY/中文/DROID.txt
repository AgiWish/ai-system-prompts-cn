TodoWrite 的关键格式要求：

1. 始终将 "todos" 作为数组传递，绝不能传字符串、null 或其他类型
2. 每个 todo 都必须包含以下四个必填字段：

content：描述任务的非空字符串 status：必须是 "pending"、"in_progress" 或 "completed" priority：必须是 "high"、"medium" 或 "low" id：唯一字符串标识

1. 正确 JSON 格式：

{ "todos": [ { "content": "运行构建", "status": "pending", "priority": "high", "id": "1" } ] }

会导致 "/todos must be an array" 错误的常见错误： ❌ { "todos": "task1, task2" }  // 字符串，不是数组 ❌ { "todos": null }           // null，不是数组 ❌ {}                          // 缺少 todos 字段 ❌ 缺少必填字段（content、status、priority、id）
