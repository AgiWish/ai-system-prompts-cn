## 风格原则 / Style Principles
1. **阅读优先**：正文区域永远是视觉重心，工具栏、状态栏、收藏按钮都必须退到低对比层级。
2. **暖白不等于复古**：使用纸感底色和衬线正文，但控件保持现代、清晰、低噪声，避免做成旧书页或仿古界面。
3. **三栏即工作台**：左侧负责资料定位，中间负责中英对照阅读，右侧负责来源、状态、笔记和术语，三栏职责不可混杂。
4. **高亮只服务对齐**：段落高亮、链接高亮、收藏高亮只在用户定位或校对时出现，默认阅读状态不应满屏色块。
5. **状态必须可信**：已校对、机器初译、仅英文、版权受限等状态使用明确语义色和文字，不用装饰性标签替代。
6. **书籍与 Prompt 共用骨架**：书籍章节和 AI Prompt 文件都进入同一个阅读器，但用标签和元信息区分资料类型。
7. **低饱和少色相**：主色只保留青绿、铜红、金色、技术蓝四类，小面积使用；大面积背景只用暖白与纸灰。
8. **正文排版留呼吸**：长文阅读字号不低于 `16px`，推荐 `18px / 1.78`，段落间距大于普通文档站。
9. **组件几何克制**：卡片圆角控制在 `8px`–`16px`，避免消费 App 式大圆角和过度软萌。
10. **工具栏密度高但不拥挤**：视图切换、同步滚动、章节跳转可以密集排列，但要用分组、分隔线和状态色保证可扫读。
11. **深色只给代码和原始片段**：大面积暗色会降低书籍阅读舒适度，黑底只用于代码、原始 Prompt 块或特殊预览。
12. **动效服务定位**：只使用短过渡、轻微高亮淡入、目录定位，不做滚动视差、大面积动效或营销式入场动画。

## 提示词包
### 基础提示词

**审美方向**
为“AI 资料中英对照阅读平台”生成一个 Web 界面，风格是**暖白纸感的技术档案阅读工作台**。界面需要同时承载 AI Prompt 文件和经典书籍章节，核心体验是中英双栏对照、章节导航、翻译状态、来源说明和笔记沉淀。整体应安静、可信、长文友好，避免营销站、暗色赛博和过度 AI 渐变。

**Token 约束**
使用暖白页面底色 `#F7F3EA`，正文表面 `#FFFCF6`，面板表面 `#F1E9DD`，主文字 `#241B16`，次级文字 `#6B6157`。主品牌色为沉稳青绿 `#1E635A`，用于已校对、可读和主按钮；铜红 `#A94F2D` 用于当前选中、来源提醒和关键状态；金色 `#C7A148` 用于收藏、书籍经典感和段落摘录；技术蓝 `#365F8C` 只用于 AI / Prompt 标签。正文中文优先使用 `"Source Han Serif SC", "Noto Serif SC", "Songti SC"`，英文阅读使用 `"Iowan Old Style", "Charter", "Georgia"`，UI 控件使用 `"Inter", "PingFang SC"`。正文推荐 `18px / 1.78`，控件 `14px / 1.45`。

**布局约束**
桌面采用三栏工作台：左侧 `240px` 资料目录，中间 `minmax(0, 1fr)` 中英双栏正文，右侧 `300px` 信息面板。中间正文在双栏模式下使用 `1fr 1fr`，中文聚焦模式为 `1.35fr 0.65fr`。顶部工具栏高度约 `56px`，提供资料类型、标题、搜索、视图切换、同步滚动和设置入口。移动端折叠为单栏阅读，目录与信息面板通过抽屉进入。页面间距遵循 `4/8/12/16/20/24/32/40px` 节奏。

**交互约束**
段落 hover 或点击时，用 `rgba(199, 161, 72, 0.18)` 做当前段落高亮；中英链接段落用 `rgba(30, 99, 90, 0.12)` 做同步提示。按钮悬停只改变底色、边框或轻微投影；按下使用 `translateY(1px)`；聚焦使用 `0 0 0 3px rgba(169, 79, 45, 0.16)`。资料读取失败、无中文译文、版权受限必须显示明确空状态，不允许空白。

**避免项**
- 不要使用大面积紫色、蓝紫渐变、霓虹光效或“AI 炫技”背景。
- 不要把首页做成营销 landing page，第一屏应直接进入资料库或阅读工作台。
- 不要使用纯黑背景作为主阅读环境。
- 不要把书籍卡片做成过度拟物书封，也不要用大圆角消费 App 卡片。
- 不要在正文区域放大面积阴影、强边框或装饰图。
- 不要让工具栏按钮抢正文视觉重心。
- 不要混用过多字体；正文衬线、UI 无衬线、代码等宽三类足够。

### 变体提示词

**更偏书籍阅读**
保持三栏布局不变，将正文表面提高到 `#FFFEFA`，正文中文使用更强的宋体 / 思源宋体，段落行高提升到 `1.86`，减少工具栏按钮数量，右侧信息面板默认折叠。保留青绿主色与铜红选中态，不引入新主色。

**更偏技术文档**
保持暖白背景，但将 UI 字体权重提高半级，正文默认使用无衬线，代码块和 Prompt 原文块比例增加。技术蓝 `#365F8C` 的使用面积可略增，但不要超过页面强调色的 `20%`。左侧目录密度提高，卡片投影降低。

**更偏校对工作台**
中英双栏宽度固定为 `1fr 1fr`，段落编号、区块数量、对齐状态更显眼。当前段落高亮从金色浅底改为青绿浅底，右侧面板优先展示“区块数不一致”“待校对”“已确认”等状态。整体颜色不变，只增强状态可见性。

**更轻量首页**
资料库首页可使用卡片网格，但不要做大 Hero。顶部只保留项目名、搜索和分类筛选；主内容按“AI Prompt / 经典书籍 / 最近更新 / 待校对”分区。书籍卡片可使用极小书脊色条，Prompt 卡片使用厂商字母标识。

## 复用说明
- **必须保持不变**：暖白纸感底色、三栏阅读工作台、正文高可读排版、青绿/铜红/金色/技术蓝的低饱和色彩体系、状态标签文字化、中英段落同步高亮。
- **可弹性调整**：正文衬线或无衬线的权重、右侧信息面板是否默认展开、书籍卡片密度、工具栏按钮数量、纸纹背景透明度。
- **风格跑偏风险**：使用暗色赛博背景、紫蓝 AI 渐变、营销首屏、大面积插画、过度圆角、强阴影或过多装饰图标，都会让产品从“阅读档案工具”偏成普通 AI SaaS 落地页。
