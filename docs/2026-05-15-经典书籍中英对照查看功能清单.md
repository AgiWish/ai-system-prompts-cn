# 经典书籍中英对照查看功能清单

## 1. 假设与待确认

- 产品定位：在当前「AI 系统提示词 / AI 产品资料库」基础上，新增一个经典书籍阅读模块，让用户按书籍、章节、主题查看英文原文与中文翻译。
- 首个种子内容：根目录 `创始人手册-构建一个AI原生初创公司.pdf`，后续可扩展到创业、产品、AI、管理、认知等经典书籍。
- 产品形态：优先做 Web 静态原型或轻量 Web 应用；移动端适配作为后续，不先做独立 App。
- 内容维护方式：MVP 可以先用结构化 Markdown / JSON 手工录入，后续再做 PDF / EPUB 导入、自动分章、机器翻译、人工校对。
- 中英对照颗粒度：MVP 以「段落级」对照为主，先不追求逐句级精细对齐。
- 合规边界：仅录入有授权、公开许可、个人学习笔记或内部合法使用范围内的内容；版权状态需要逐本确认。
- 待确认：这套功能是放进现有 `index.html` 资料站，还是作为新的 `web/book-library` 原型模块独立呈现？
- 待确认：用户更需要「完整阅读」还是「章节摘要 + 金句 + 原文对照」？
- 待确认：中文翻译来源是人工翻译、AI 初译后人工校对，还是只做机器辅助阅读？

### 1. 经典书籍库首页

- **入口与导航**：站点主导航新增「经典书籍」入口，默认进入书籍库。
- **页面结构（自上而下）**：顶部导航 → 分类筛选 → 推荐/最近更新 → 书籍卡片列表 → 底部状态说明。
- **主要功能**：WEB-001 查看书籍列表，WEB-002 按主题/语言/状态筛选，WEB-003 查看最近更新与推荐书籍。
- **备注**：P0；首期至少展示《创始人手册》和 3-5 本占位经典书籍，避免页面像单本书工具。
- **文件名称（建议）**：book-library

### 2. 书籍详情页

- **入口与导航**：从书籍卡片进入；可返回书籍库。
- **页面结构（自上而下）**：书籍标题与封面信息 → 简介/适合人群 → 章节目录预览 → 阅读入口 → 来源与版权说明。
- **主要功能**：WEB-010 查看书籍元信息，WEB-011 查看章节目录，WEB-012 从上次阅读位置继续。
- **备注**：P0；需要明显标注内容状态，如「已校对」「机器初译」「仅英文」。
- **文件名称（建议）**：book-detail

### 3. 中英对照阅读器

- **入口与导航**：从书籍详情、章节目录、搜索结果、收藏条目进入。
- **页面结构（自上而下）**：顶部阅读栏 → 左侧章节目录/进度 → 中部双栏段落对照 → 右侧笔记/术语辅助 → 底部上一节/下一节。
- **主要功能**：WEB-020 双栏阅读，WEB-021 中英/英文/中文视图切换，WEB-022 章节跳转，WEB-023 段落定位与高亮。
- **备注**：P0；这是核心页面，视觉重心应在段落对照本身，而不是工具按钮。
- **文件名称（建议）**：bilingual-reader

### 4. 章节目录与进度页

- **入口与导航**：书籍详情页内进入，也可由阅读器左侧目录展开。
- **页面结构（自上而下）**：章节列表 → 每章状态 → 阅读进度 → 继续阅读按钮。
- **主要功能**：WEB-030 查看全书章节结构，WEB-031 标记阅读进度与完成状态。
- **备注**：P1；MVP 可并入书籍详情页，单独页面作为下一阶段增强。
- **文件名称（建议）**：chapter-progress

### 5. 全文搜索结果页

- **入口与导航**：顶部全局搜索或书内搜索进入。
- **页面结构（自上而下）**：搜索框 → 范围筛选 → 结果列表 → 命中段落预览 → 跳转阅读器。
- **主要功能**：WEB-040 按关键词搜索书籍/章节/段落，WEB-041 从命中结果跳转到原文位置。
- **备注**：P1；MVP 可先做书名与章节搜索，全文段落搜索后置。
- **文件名称（建议）**：book-search

### 6. 笔记与收藏页

- **入口与导航**：阅读器右侧笔记区、个人入口进入。
- **页面结构（自上而下）**：收藏筛选 → 段落摘录列表 → 用户笔记 → 回到原文按钮。
- **主要功能**：WEB-050 收藏段落/金句，WEB-051 为段落添加个人笔记。
- **备注**：P1；如果首期只做静态原型，可先表现收藏状态，不做真实账号同步。
- **文件名称（建议）**：reading-notes

### 7. 内容录入管理页

- **入口与导航**：内容维护者从管理入口进入；普通读者不可见。
- **页面结构（自上而下）**：新增书籍表单 → 文件/文本导入 → 元信息填写 → 章节拆分预览 → 发布状态。
- **主要功能**：ADM-010 新增书籍元信息，ADM-011 导入 PDF/Markdown/文本，ADM-012 生成章节草稿。
- **备注**：P1；MVP 可以不做真实导入，只用它表达未来录书链路。
- **文件名称（建议）**：book-ingest

### 8. 段落对齐校对页

- **入口与导航**：从内容录入管理页进入某一本书的校对任务。
- **页面结构（自上而下）**：章节选择 → 原文段落列表 → 中文译文列 → 对齐/合并/拆分操作 → 校对状态。
- **主要功能**：ADM-020 查看原文与译文对齐状态，ADM-021 调整段落对齐，ADM-022 标记校对完成。
- **备注**：P1；如果中英质量是产品核心，建议这一页不要过晚才做。
- **文件名称（建议）**：alignment-review

### 9. 阅读偏好设置页

- **入口与导航**：阅读器工具栏进入。
- **页面结构（自上而下）**：显示模式 → 字号/行高 → 双栏宽度 → 术语辅助开关 → 保存。
- **主要功能**：WEB-060 调整阅读显示，WEB-061 保存个人阅读偏好。
- **备注**：P2；MVP 可仅在阅读器顶部提供快速切换。
- **文件名称（建议）**：reading-settings

### 10. 空状态与版权提示页

- **入口与导航**：无书籍、无章节、版权未确认、翻译未完成时展示。
- **页面结构（自上而下）**：状态标题 → 原因说明 → 可执行下一步 → 返回入口。
- **主要功能**：WEB-070 展示内容不可读原因，WEB-071 引导返回书籍库或申请录入。
- **备注**：P0；内容型产品必须避免用户点进来看到空白。
- **文件名称（建议）**：book-empty-state

## 4. 结构化详细功能列表

### 书籍发现

#### 书籍库

- **WEB-001** 读者在书籍库查看所有已录入书籍，快速知道有哪些经典书可读。
- **WEB-002** 读者按主题、语言状态、校对状态筛选书籍。
- **WEB-003** 读者查看推荐书籍与最近更新，发现新增内容。

#### 书籍详情

- **WEB-010** 读者查看书名、作者、简介、主题标签、内容状态。
- **WEB-011** 读者查看章节目录，判断是否从某章开始阅读。
- **WEB-012** 读者从上次阅读位置继续进入阅读器。

### 对照阅读

#### 阅读器主体

- **WEB-020** 读者按段落查看英文原文与中文译文双栏对照。
- **WEB-021** 读者切换「中英对照 / 仅英文 / 仅中文」视图。
- **WEB-022** 读者在阅读器中快速切换章节。
- **WEB-023** 读者点击搜索结果或目录后定位到具体段落，并看到高亮。

#### 阅读进度

- **WEB-030** 读者查看全书章节与当前阅读进度。
- **WEB-031** 读者看到章节是否已读、未读、读到一半。

### 检索与知识沉淀

#### 搜索

- **WEB-040** 读者按关键词搜索书名、章节名、段落内容。
- **WEB-041** 读者从搜索结果直接跳转到对应段落位置。

#### 收藏与笔记

- **WEB-050** 读者收藏重要段落、金句或概念。
- **WEB-051** 读者为段落添加个人笔记，并能从笔记页回到原文。

### 内容维护

#### 书籍录入

- **ADM-010** 内容维护者新增书籍元信息，包括书名、作者、来源、版权说明、主题标签。
- **ADM-011** 内容维护者导入 PDF、Markdown 或纯文本作为书籍原文。
- **ADM-012** 内容维护者将导入内容拆分为章节草稿。

#### 对齐校对

- **ADM-020** 内容维护者查看英文原文与中文译文是否完成段落级对齐。
- **ADM-021** 内容维护者调整段落拆分、合并、排序与译文匹配。
- **ADM-022** 内容维护者标记章节为「机器初译」「人工校对中」「已校对」。

### 阅读偏好与异常状态

#### 设置

- **WEB-060** 读者调整字号、行高、双栏宽度。
- **WEB-061** 读者保存阅读偏好，下次进入继续沿用。

#### 状态提示

- **WEB-070** 系统在无内容、无翻译、无授权、解析失败时展示明确原因。
- **WEB-071** 系统给出返回书籍库、查看英文原文、申请录入等下一步入口。

## 6. 分期

### MVP

- 书籍库首页：展示《创始人手册》和经典书籍占位卡片。
- 书籍详情页：展示简介、章节目录、阅读入口、内容状态、版权说明。
- 中英对照阅读器：支持段落级双栏阅读、视图切换、章节跳转。
- 空状态与版权提示：无翻译、无授权、无章节时有清晰提示。
- 内容数据：先用 1 本书的结构化章节和少量样例段落打通体验，不先追求全书完整录入。

### 下一阶段

- 全书章节进度、继续阅读、书内搜索。
- 段落收藏、金句列表、个人笔记。

## 7. 范围与体验约束

- 阅读器要克制：主视觉只服务于阅读，不做过多装饰卡片和营销文案。
- 中英文排版要等高或接近等高，避免读者无法判断哪段对应哪段。
- 每本书必须有内容可信度状态，不能让用户误以为机器初译就是正式出版译本。
- 导入链路可以晚做，但内容结构必须从一开始支持多书、多章节、多段落。
- 版权说明必须成为书籍详情页的一等信息，不放在页面底部小字里弱化。
