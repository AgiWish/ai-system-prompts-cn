# 输出模板（参考）

复制后按需改写。将 `[方括号]` 占位替换为实际内容。本技能偏 **原型与范围**，列名避免开发/测试术语。

## 图例块（全文放一次）

```markdown
**优先级说明（MoSCoW）**

- **必须有**：没有则 MVP 原型/范围不成立
- **应该有**：下一版原型或迭代优先考虑
- **可以有**：锦上添花
- **本阶段不做**：明确排除

**ID 前缀（便于对照页面与分期）**：`WEB-` 官网 · `APP-` 产品内 · `ADM-` 管理端 · `INT-` 外部能力依赖（仅记范围） · `NFR-` 尽量少用，仅记影响原型的约束
```

## 完整页面结构列表（必选，交付第 2 章；**禁止**宽表承载下列五项）

每屏 **单独一小节** + 无序列表（入口与导航、页面结构、主要功能、备注、文件名称建议）。可选 **2 列窄表** 仅作「序号｜页面名称」目录。

```markdown
## 完整页面结构列表

### 页面一览（可选）

| 序号 | 页面 / 屏幕名称（建议） |
|------|-------------------------|
| 1 | 登录 |
| 2 | 任务首页 |
| … | … |

### 1. 登录

- **入口与导航**：`/login`，未登录可进
- **页面结构（自上而下）**：顶栏 Logo → 表单区 → 辅助链
- **主要功能**：`APP-001`（登录）, `APP-001`（忘记密码入口）
- **备注**：访客；P0
- **文件名称（建议）**：`login`（或 `login-screen`，勿加 `Scr-xx-` 前缀）

### 2. 任务首页

- **入口与导航**：Tab「任务」、登录后默认
- **页面结构（自上而下）**：顶栏 → 筛选 → 列表 → 悬浮新建
- **主要功能**：`APP-010`, `APP-020`
- **备注**：P0
- **文件名称（建议）**：`task-home`
```
