基础提示词 / Foundation Prompt

构建一个营销型 SaaS 落地页视觉基座，风格为「暖米白玻璃拟态 + 橙色点缀 + 大字号极简排版」。

严格使用以下约束：

  • 背景使用 #f5efe6，并加入两层柔和模糊光斑（暖橙 + 白色）。
  • 表面层使用半透明白（约 rgba(255,255,255,0.48~0.62)）与 12px~14px blur。
  • 主文案颜色使用深暖灰 #2f2a25，辅助文案 #645d56。
  • 主按钮使用 #f28a2b，次按钮为半透明白底。
  • 标题层级：Hero clamp(44px, 7vw, 72px)，区块标题 40px，卡片标题 22px，正文 14~17px。
  • 容器最大宽度 1160px，区块上下间距 80px。
  • 卡片圆角 18px，输入框圆角 14px，标签胶囊 999px。
  • 动效采用克制策略：允许轻微呼吸感，禁止 hover 上浮或强位移。

避免：

  • 高饱和蓝紫渐变主导画面
  • 过重黑色描边与硬阴影
  • 多套圆角体系混用
  • 按钮与卡片出现明显「漂浮跳动」悬停动画

组件提示词 / Component Prompt

在上述视觉 DNA 下生成组件：导航、CTA 按钮、功能卡、价格卡、FAQ、表单与 Element Plus 下拉。

组件约束：

  • 按钮统一高度 46px，主次按钮视觉对比明确但不过度夸张。
  • 卡片统一采用浅透明白底 + 细边线 + 软阴影；深色卡仅用于「推荐方案」等重点。
  • 表单输入框去传统边框，使用圆角背景与内阴影；focus 用橙色弱高亮。
  • 下拉面板（Element Plus）需与表单一致：半透明米白背景、14px 圆角、柔和阴影。
  • FAQ 折叠仅做图标旋转与展开收起，不加入弹跳位移动效。
  • 指标卡支持一行平铺（桌面 4 列，平板 2 列，手机 1 列）。

避免：

  • 输入框深色描边
  • 下拉面板使用纯白硬边弹层
  • 卡片间距不一致
  • 文案层级全部同号同粗

变体提示词 / Variation Prompt

保持同一风格 DNA，生成「更沉稳商务版」变体：

  • 维持暖米白基底与玻璃质感不变；
  • 将强调色从 #f28a2b 稍降饱和至 ~#de8b42；
  • 卡片透明度从 0.55 降至 0.50，阴影轻微减弱；
  • 标题字重保持 700，正文增加留白与行距，信息密度再降低约 10%；
  • 所有交互保持稳态（hover 不上浮，仅 focus 高亮）。

避免：

  • 改成冷色科技蓝主调
  • 删除玻璃层导致风格失真
  • 增加强对比霓虹或重纹理背景
